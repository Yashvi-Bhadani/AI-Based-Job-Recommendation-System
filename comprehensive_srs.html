1. Introduction

1.1 Purpose

The purpose of this document is to detail the software requirements for the AI-Based Job Recommendation System (AI Job Matcher). It provides a comprehensive overview of the system's intended capabilities, features, and constraints. This document serves as a guideline for developers, testers, and stakeholders throughout the development lifecycle, moving from an initial prototype to a robust, enterprise-grade application structure.

1.2 Scope

The AI Job Matcher is a scalable web-based application designed to bridge the gap between job seekers and employers by leveraging advanced Artificial Intelligence. The system parses uploaded resumes, evaluates candidate skills and experience using Natural Language Processing (NLP) techniques, and recommends the most suitable job descriptions. It streamlines the recruitment process, substantially reducing manual reviewing time and increasing matching accuracy.

2. Overall Description

2.1 Product Perspective

The AI Job Matcher operates as a distributed web application adopting a MERN-stack methodology (MongoDB, Express, React, Node.js). It consists of a React-based frontend client decoupled from a Node.js/Express.js backend server. The backend interfaces with a MongoDB database for scalable, persistent data storage and orchestrates calls to internal or external AI models to perform its core analytical recommendation tasks.

2.2 Product Features

  • User Management: Secure registration, authentication, and role-based access control (differentiating Admin, Job Seeker, and Employer capabilities).
  • Intelligent Resume Parsing: Automated, AI-driven extraction of skills, experience, and educational background from raw PDF or document uploads.
  • Job Lifecycle Management: Comprehensive tools for Employers to post, edit, pause, and manage job listings.
  • AI Job Recommendation Engine: The proprietary system that intelligently correlates and matches job seekers with highly relevant job postings based on parsed contextual data, providing a percentage match score.
  • Application Tracking Workflow: A seamless interface where users can track the real-time status of their applications, while employers manage received applicant pipelines.
  • Admin Dashboard: A centralized control panel for overarching platform administration, user management, and system analytics.

2.3 User Classes and Characteristics

  • Job Seekers: Users looking for employment. They have technical proficiency ranging from low to high. They expect an intuitive interface to upload their resumes and view parsed results effortlessly.
  • Employers/Recruiters: Professional users who manage job postings and review AI-sorted candidate pipelines. Require rapid access to data and clear analytical matching scores.
  • Administrators: Technical personnel managing system health, user roles, and configuring system-wide parameters (e.g., AI model thresholds).

2.4 Operating Environment

  • Client/Frontend: Modern Web Browsers supporting HTML5 and ES6 JavaScript (Google Chrome 90+, Mozilla Firefox 88+, Safari 14+, MS Edge 90+).
  • Server Processing: Node.js runtime environment (v18.x or newer).
  • Database: MongoDB Atlas (Cloud) or localized MongoDB instances (v5.0+).
  • Hosting Ecosystem: Cross-platform compatibility enabled via containerization (Docker) extending deployment to AWS, Azure, or standard VPS configurations running Linux.

3. System Features Detailed Specification

3.1 Authentication & Security (Managed by Astha & Manthan)

Description: Secure access protocols governing entry to the system through credential verification and robust session management.

  • Functional Requirements:
    • System shall allow users to securely register and authenticate via email and password combination.
    • System must encrypt user passwords upon creation utilizing bcrypt algorithm with a minimum salt round of 10.
    • Upon successful authentication, the system shall issue a JSON Web Token (JWT) with an encoded expiration timeframe to maintain a stateless session.
    • The server infrastructure shall employ stringent CORS policies mapped only to authorized frontend origins.

3.2 AI Recommendation Engine (Managed by Yashvi)

Description: The core intelligent architectural module that ingests unformatted text and algorithmically bridges resumes to structured job requirements.

  • Functional Requirements:
    • The endpoint shall receive resume text or binary file streams and accurately extract predefined entities utilizing NLP pipelines.
    • The recommendation matrix shall score job postings individually against parsed resume profiles, yielding a quantifiable match percentage evaluating semantic similarity of requirements.
    • The module must present the top $N$ matching jobs to the user efficiently, ordered by relevance.

3.3 Application & Job Management Workflow (Backend by Manthan, UI by Shivansh)

Description: The CRUD operational workflows handling the expansive lifecycle of job postings alongside candidate submittals.

  • Functional Requirements:
    • Authenticated Employers shall be granted access to create, read, update, and gracefully delete (or archive) job postings.
    • Authenticating Job seekers shall be empowered to execute application workflows against active job postings.
    • System architecture shall maintain atomic state updates corresponding to application progression (e.g., Pending, Under Review, Interviewing, Accepted, Rejected).
    • The User Interface shall dynamically render responsive dashboards uniquely tailored to the authenticated user's assigned role and active states.

4. Non-Functional Requirements

4.1 Performance Requirements

  • Standard transactional API response times (CRUD operations excluding file parsing) should optimally resolve under 300ms at the 95th percentile under normal load.
  • The AI parsing and matching process must operate asynchronously without blocking the main event Node loop, returning processing statuses or webhook triggers for long-running analyses exceeding 5000ms.

4.2 Security Requirements

  • Encryption in transit: All sensitive authentication and data payloads must be fundamentally transmitted over TLS/HTTPS.
  • Data hygiene: Database connection strings, cryptographic JWT secrets, and external API keys must be isolated securely utilizing environmental configuration techniques (`.env` variables) and never committed to source version control.
  • Passwords must inherently be salted and hashed at rest; they must never remain stored or logged in plain cryptographic text.

4.3 Software Quality Attributes

  • Reliability: The backend architecture shall incorporate comprehensive centralized error handling middlewares to gracefully trap unhandled promise rejections or exceptions, mitigating hard internal server crashes.
  • Maintainability: Codebases must adhere to modular, component-based scaling rules on both Frontend (React components) and Backend (isolated Route/Controller/Model paradigms) to ensure swift onboarding of progressive features.

5. User Interfaces & Interactions (Frontend by Shivansh)

The User Experience language focuses on modern, sleek, and intuitive visual hierarchies engineered with Tailwind CSS constraints.

  • Role-Based Central Dashboards: Dedicated graphical views isolating operational concerns. Employers see posting metrics and raw applicant funnels; Job Seekers interact with match listings and application status timelines.
  • Dynamic Job Feed: An engaging, elegantly paginated (or infinite-scroll capable) list presenting AI-matched job postings categorized by precision match scores alongside pertinent metadata tags.
  • Profile Configuration Hub: A consolidated interface module reserved for defining granular user details, securely uploading resume attachments, and adjusting notification presets.
